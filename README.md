Headroom compresses everything your AI agent reads вҖ” tool outputs, logs, RAG chunks, files, and conversation history вҖ” before it reaches the LLM. Same answers, fraction of the tokens.

## What it does

- **Library** вҖ” `compress(messages)` in Python, inline in any app
- **MCP server** вҖ” `headroom_compress`, `headroom_retrieve`, `headroom_stats` for any MCP client
- **Reversible (CCR)** вҖ” originals are cached for retrieval on demand

## How it works (30 seconds)

```
 Your agent / app
   (Claude Code, Cursor, Codex, LangChain, Agno, Strands, your own codeвҖҰ)
        в”Ӯ   prompts В· tool outputs В· logs В· RAG results В· files
        в–ј
    в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ

    в”Ӯ  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ  в”Ӯ

    в”Ӯ  Reversible CCR store  В·  MCP server               в”Ӯ
    в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

- **ContentRouter** вҖ” detects content type, selects the right compressor
- **SmartCrusher / Kompress-v2-base** вҖ” compress JSON, or prose and source code
- **CacheAligner** вҖ” stabilizes prefixes so provider KV caches actually hit
- **CCR** вҖ” stores originals locally; LLM calls `headroom_retrieve` if it needs them

вҶ’ [Kompress-v2-base model card](https://huggingface.co/chopratejas/kompress-v2-base)

## Get started (60 seconds)

```bash
# 1 вҖ” Install
pip install "headroom-ai[all]"          # everything
# or: pip install "headroom-ai[mcp]"     # just the MCP server
```

```python
# 2 вҖ” Compress inline in any Python app
from headroom import compress

result = compress(messages, model="claude-sonnet-4")
# result.messages  вҶ’ compressed; CCR keeps originals retrievable
```

```bash
# 3 вҖ” Or run the MCP server for Claude Code / Cursor / any MCP host
python -m headroom.ccr.mcp_server       # exposes headroom_compress / _retrieve / _stats
```

Granular extras: `[mcp]`, `[ml]` (Kompress-v2-base), `[html]`, `[dev]`. Requires **Python 3.10+**.

## Proof

**Token reduction on real captured data** вҖ” reproducible; inputs committed under `benchmarks/data/`, captured by `benchmarks/run_bench.py` into [BASELINE.md](benchmarks/BASELINE.md):

| Dataset | Items | Before | After  | Reduction       | Info retention |
|---------|------:|-------:|-------:|-----------------|---------------:|
| search  |    90 |  4,102 |  2,462 | 40% (lossless)  |           100% |
| logs    |    90 |  8,595 |  1,332 | 84%*            |   100% (via CCR) |
| code    |     7 | 41,025 | 41,025 | 0% (passthrough)|           100% |

<sub>*log savings come partly from row deletion, not free compression вҖ” every dropped row stays CCR-recoverable within the configured TTL. `code` is large distinct source files that don't compress, so Headroom passes them through untouched (0%).</sub>

The table above is a single conservative capture (lossless structural folding, minimal offload). The **60вҖ“95%** headline reflects the fuller 6-seed adversarial sweep вҖ” logs, search, and disk workloads land 80вҖ“95% with CCR offload, with honest per-workload caveats (code passes through at 0%, low-redundancy multiturn lands lower). Full methodology and re-runnable sweeps: [BENCHMARKS.md](BENCHMARKS.md).

## When to use В· When to skip

**Great fit if youвҖҰ**
- feed large tool outputs, logs, RAG chunks, or files into an LLM and want fewer tokens
- want compression you can drop into any Python app, or expose to an MCP host
- need reversible compression вҖ” originals are retrievable via CCR within the configured TTL

**Skip it if youвҖҰ**
- only use a single provider's native compaction and don't process large external context
- can't run the compression locally in your own process

<details>
<summary><b>What's inside</b></summary>

- **SmartCrusher** вҖ” universal JSON: arrays of dicts, nested objects, mixed types.
- **Kompress-v2-base** вҖ” our HuggingFace model, trained on agentic traces; also handles source code.
- **SearchCompressor / LogCompressor / DiffCompressor** вҖ” search results, build logs, and diffs.
- **HTMLExtractor** вҖ” strips boilerplate from fetched HTML (optional, needs `trafilatura`).
- **CacheAligner** вҖ” stabilizes prefixes so Anthropic/OpenAI KV caches actually hit.
- **CCR** вҖ” reversible compression; LLM retrieves originals on demand.

</details>

<details>
<summary><b>Pipeline internals</b></summary>

`compress()` emits three compression lifecycle stages:

`Input Received` вҶ’ `Input Routed` вҶ’ `Input Compressed`

- **Transforms** do the work: CacheAligner, CrossMessageDeduper, ContentRouter, SmartCrusher, Kompress-v2-base.
- **Pipeline extensions** observe or customize these stages via `on_pipeline_event(...)`; `compress()` passes your `hooks` object as the extension.
- **Compression hooks** sit alongside the lifecycle as an additional extension seam.

</details>

## Install

```bash
pip install "headroom-ai[all]"          # everything
```

Granular extras: `[mcp]` (MCP server), `[ml]` (Kompress-v2-base), `[html]` (HTML extraction), `[dev]`. Requires **Python 3.10+**.

Using `pipx`? Choose a supported interpreter explicitly:

```bash
pipx install --python python3.13 "headroom-ai[all]"
```

### Corporate / SSL-inspection environments

If `pip install "headroom-ai[all]"` fails with `CERTIFICATE_VERIFY_FAILED`
(`unable to get local issuer certificate`), your network uses **SSL inspection** вҖ” a MITM
proxy presenting a company-issued CA. The build backend (`maturin`) downloads `rustup` over a
connection your TLS stack doesn't trust. **Install Rust first** so the build doesn't fetch it:

```bash
# macOS / Linux
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh && rustup default stable
# Windows
winget install Rustlang.Rustup && rustup default stable
```

Restart your shell, then `pip install "headroom-ai[all]"`. A prebuilt wheel avoids the Rust
build entirely where available: `pip install --only-binary headroom-ai headroom-ai`.

Two runtime assets are fetched over TLS; if they are blocked, trust your corporate CA via
`REQUESTS_CA_BUNDLE` / `SSL_CERT_FILE` / `CURL_CA_BUNDLE`:

- **`cdn.pyke.io`** вҖ” the ONNX Runtime for the Rust core. Alternatively pre-provide it with
  `ORT_STRATEGY=system` and `ORT_LIB_LOCATION=/path/to/onnxruntime`.
- **`huggingface.co`** вҖ” the `chopratejas/kompress-v2-base` compression model. Pre-download it and run with
  `HF_HUB_OFFLINE=1`, or set `HF_ENDPOINT` to a trusted mirror.

Running with compression disabled requires neither asset.

Headroom runs **locally**, covers **every** content type, and is **reversible**.

|                                                                              | Scope                                          | Deploy                             | Local | Reversible |
|------------------------------------------------------------------------------|------------------------------------------------|------------------------------------|:-----:|:----------:|
| **Headroom**                                                                 | All context вҖ” tools, RAG, logs, files, history | library В· MCP                      | Yes   | Yes        |
| [RTK](https://github.com/rtk-ai/rtk)                                        | CLI command outputs                            | CLI wrapper                        | Yes   | No         |
| [lean-ctx](https://github.com/yvgude/lean-ctx)                               | CLI commands, MCP tools, editor rules          | CLI wrapper В· MCP                  | Yes   | No         |
| [Compresr](https://compresr.ai), [Token Co.](https://thetokencompany.ai)    | Text sent to their API                         | Hosted API call                    | No    | No         |
| OpenAI Compaction                                                            | Conversation history                           | Provider-native                    | No    | No         |

> **RTK** ([rtk-ai/rtk](https://github.com/rtk-ai/rtk)) is a complementary CLI-output rewriter вҖ” a peer in the table above, **not** bundled with or a dependency of Headroom. If you already use it for shell-output rewriting, Headroom compresses everything downstream; the two compose cleanly. Credit to the RTK team for a great tool.
